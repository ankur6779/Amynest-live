# AmyNest — Capacitor iOS + Android Setup Guide

This Capacitor project wraps the `kidschedule` PWA into a **native iOS (Xcode)** and **native Android** shell.

---

## Architecture

```
amynest-capacitor/
├── capacitor.config.ts       ← Capacitor configuration
├── www/                      ← Built web assets (populated by build:web script)
├── ios/                      ← Generated by: npx cap add ios   (macOS only)
├── android/                  ← Generated by: npx cap add android
├── ios-config/
│   ├── Info-permissions.plist    ← Paste into ios/App/App/Info.plist
│   └── AmyNest.entitlements      ← Replace ios/App/App/App.entitlements
└── android-config/
    ├── AndroidManifest-additions.xml
    └── network_security_config.xml
```

---

## Prerequisites

### For iOS (requires a Mac)
- macOS 13+ (Ventura or later)
- Xcode 15+ (download from App Store)
- CocoaPods: `sudo gem install cocoapods`
- Node.js 20+, pnpm 9+

### For Android
- Android Studio Hedgehog (2023.1) or later
- Android SDK API 22+
- Java 17+ (bundled with Android Studio)
- Node.js 20+, pnpm 9+

## Step-by-Step Setup

### 1. Clone and install dependencies

```bash
git clone <your-repo-url>
cd <repo-root>
pnpm install
```

### 2. Build the web app

From the **repo root**:

```bash
cd artifacts/amynest-capacitor

# Copy your Firebase + environment variables first:
export FIREBASE_API_KEY="your-key"
export FIREBASE_AUTH_DOMAIN="your-project.firebaseapp.com"
export FIREBASE_PROJECT_ID="your-project"
export FIREBASE_APP_ID="1:xxx:web:xxx"
export FIREBASE_MESSAGING_SENDER_ID="xxx"
export VITE_FIREBASE_VAPID_KEY="xxx"

pnpm run build:web
```

This will:
1. Build `artifacts/kidschedule` with `BASE_PATH=/` (standalone mode)
2. Copy the output into `www/`

---

### 3a. iOS Setup (run on Mac only)

```bash
cd artifacts/amynest-capacitor

# Add the iOS platform (generates ios/ folder)
npx cap add ios

# Install CocoaPods dependencies
cd ios/App && pod install && cd ../..

# Sync web assets + plugins into the Xcode project
npx cap sync ios

# Open in Xcode
npx cap open ios
```

#### Inside Xcode:
1. Select the **App** target → **General** tab
   - Bundle Identifier: `com.amynest.app`
   - Version: `1.0.0`
   - Build: `1`
   - Minimum Deployments: **iOS 14.0**
2. **Signing & Capabilities** tab
   - Select your Apple Developer Team
   - Enable: **Push Notifications**, **Background Modes** (Audio, Remote notifications, Background fetch)
   - Add **Associated Domains**: `applinks:amynest.in`
3. Open `ios/App/App/Info.plist`
   - Copy all keys from `ios-config/Info-permissions.plist` into this file
4. Replace `ios/App/App/App.entitlements` with `ios-config/AmyNest.entitlements`

#### Build IPA for TestFlight:
1. In Xcode: **Product → Archive**
2. In Organizer: click **Distribute App**
3. Choose **App Store Connect** → Next → Upload
4. In App Store Connect → TestFlight → select your build
5. Add testers and submit for review

---

### 3b. Android Setup

```bash
cd artifacts/amynest-capacitor

# Add the Android platform (generates android/ folder)
npx cap add android

# Sync web assets + plugins
npx cap sync android

# Open in Android Studio
npx cap open android
```

#### Inside Android Studio:
1. Copy `android-config/network_security_config.xml` to `android/app/src/main/res/xml/`
2. Merge permissions from `android-config/AndroidManifest-additions.xml` into `android/app/src/main/AndroidManifest.xml`
3. Add Firebase config: place `google-services.json` in `android/app/`
4. **Build → Generate Signed Bundle/APK** → Android App Bundle (AAB)
5. Upload the `.aab` to Google Play Console

## Everyday Workflow (after initial setup)

When you update the web app:

```bash
cd artifacts/amynest-capacitor

# 1. Rebuild web assets
pnpm run build:web

# 2a. Sync to iOS only
pnpm run sync:ios

# 2b. Sync to Android only
pnpm run sync:android

# 2c. Sync to both
pnpm run sync

# 3. Open native IDE
pnpm run open:ios      # opens Xcode
pnpm run open:android  # opens Android Studio
```

## Google Sign-In in WKWebView (iOS)

Firebase/Google OAuth does **not** work in a plain WKWebView.  
You have two options:

### Option A — Use the Capacitor Browser Plugin (Recommended)
```bash
npm install @capacitor/browser
npx cap sync ios
```
Then in your auth code, open the Google sign-in URL via `Browser.open()` instead of a popup/redirect. Capacitor's SFSafariViewController handles it natively.

### Option B — Use Firebase App Check + native Google Sign-In
```bash
npm install @codetrix-studio/capacitor-google-auth
npx cap sync ios
```
Follow the [Capacitor Google Auth docs](https://github.com/CodetrixStudio/CapacitorGoogleAuth).

**Required:** Add your reversed Google client ID to `Info.plist` → `CFBundleURLSchemes`.  
Get it from `GoogleService-Info.plist` as the value of `REVERSED_CLIENT_ID`.

## Push Notifications

The app already has Firebase FCM integrated in the PWA.  
For the native Capacitor shell, install the Push Notifications plugin:

```bash
npm install @capacitor/push-notifications
npx cap sync
```

Then request permission in your app:
```typescript
import { PushNotifications } from '@capacitor/push-notifications';

await PushNotifications.requestPermissions();
await PushNotifications.register();

PushNotifications.addListener('registration', ({ value: token }) => {
  // Send token to your API server
  fetch('/api/push/register', {
    method: 'POST',
    body: JSON.stringify({ token, platform: 'capacitor-ios' })
  });
});
```

## Splash Screen

The existing dark purple splash is already configured.  
For native splash images, place them at:

**iOS:** `ios/App/App/Assets.xcassets/Splash.imageset/`
- `splash.png` — 2732×2732 px (covers all iPhone + iPad sizes)
- Background color `#0a061a` matches the web splash

**Android:** `android/app/src/main/res/`
- `drawable/splash.xml` (vector or PNG)
- `drawable-*dpi/splash.png` for various densities

## Safe Areas (Notch / Dynamic Island / Home Indicator)

The web app's `index.css` already has safe-area padding via `env(safe-area-inset-*)`.
The capacitor config sets `contentInset: "always"` for iOS.

If any element is being cut off by the notch or home indicator bar, add:
```css
padding-top: env(safe-area-inset-top, 0px);
padding-bottom: env(safe-area-inset-bottom, 0px);
```

## Troubleshooting

| Problem | Fix |
|---|---|
| White screen on launch | Check `www/index.html` exists; rebuild with `pnpm run build:web` |
| Google sign-in fails | Use `@capacitor/browser` plugin for OAuth flow |
| Cookies not persisting on iOS | Ensure `CapacitorHttp: { enabled: true }` in `capacitor.config.ts` |
| Push notifications not arriving | Verify APNs certificate is uploaded in Firebase console |
| "pod install" fails | Run `sudo gem install cocoapods` and try again |
| Build fails in Xcode | Run `npx cap sync ios` then clean build (Shift+Cmd+K) |
| App clips notch content | Add `viewport-fit=cover` (already done) and `env(safe-area-inset-top)` |
