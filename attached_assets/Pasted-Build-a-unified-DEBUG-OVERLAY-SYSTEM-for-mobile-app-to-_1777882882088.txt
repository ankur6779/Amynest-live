Build a unified DEBUG OVERLAY SYSTEM for mobile app to audit:

1. Functional behavior
2. UI consistency
3. Color usage
4. Mobile vs Web parity

GOAL:
Ensure pixel-perfect UI + exact logic match with web app.

---

### 1. GLOBAL DEBUG MODE

Create:

DEBUG_MODE = true

When enabled:
- Show overlay on ALL screens
- Enable logging + comparison

---

### 2. DEBUG OVERLAY UI

Floating panel (top-right / draggable):

Sections:

[FUNCTIONAL]
- Screen Name
- API Endpoint
- Request Payload
- Response Data
- Execution Time

[STATE]
- Country
- Cuisine
- Diet Type
- Child Age

[UI AUDIT]
- Font Size
- Font Weight
- Spacing
- Component Count

[COLOR AUDIT]
- Background color
- Primary color
- Button color
- Text color

---

### 3. MOBILE vs WEB COMPARISON ENGINE

Add function:

compareState(mobile, web)

Return:

{
  "logic_match": true/false,
  "ui_match": true/false,
  "color_match": true/false,
  "content_match": true/false
}

---

### 4. COLOR AUDIT SYSTEM

Extract colors from UI components:

- background
- buttons
- text
- cards

Compare with web design tokens:

Example:

{
  "mobile": "#FFFFFF",
  "web": "#F8F8F8",
  "status": "mismatch"
}

Highlight mismatch in red.

---

### 5. UI STRUCTURE VALIDATION

Check:

- Missing components
- Extra components
- Order mismatch

Example:

mobile: [header, card, button]
web:    [header, button]

→ mismatch detected
